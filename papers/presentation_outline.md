# Academic Presentation Outline

## Communication goal

By the end, reviewers should understand that the contribution is not merely a DM8 driver integration: it is a reusable database-adapter architecture validated through real Workflow/API execution and recoverable infrastructure.

## Narrative

1. Operational database access is useful but dangerous in AI workflows.
2. A single Tool contract plus Adapter Registry isolates vendor differences.
3. Shared validation protects the read-only boundary.
4. Real DM8 Workflow/API execution demonstrates the full chain.
5. 45/0/0 and cold boot turn a demo into repeatable evidence.
6. Productization closes public screenshot review while preserving honest follow-ups: portability, licensing, CI, and official Marketplace submission.
7. Phase 10 can add KingbaseES without destabilizing v1.0.

Use 12–14 slides for a 20-minute defense; reserve questions for architecture trade-offs, security scope, and experimental validity.
