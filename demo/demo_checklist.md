# Demo Checklist

## Before the room opens

- [ ] Run `dify_preflight.ps1`; do not restart unless required.
- [ ] Confirm plugin 0.1.1, Provider, and published Workflow exist.
- [ ] Confirm DM8 is reachable with `SELECT 1`.
- [ ] Inject a newly valid Workflow API key without typing it into command history.
- [ ] Run Workflow-only verification once; keep the full 45/0/0 screenshot as fallback.
- [ ] Set display to 1920×1080 and browser fullscreen.
- [ ] Close email, chat, password manager, notifications, and unrelated tabs.
- [ ] Ensure no host, username, password, token, API key, user ID, or internal URL appears.
- [ ] Open Dashboard, Plugin page, Workflow, architecture, and terminal in presentation order.

## During the demo

- [ ] Never open API Access or reveal the Key.
- [ ] Never open the Provider credential fields on the projected screen.
- [ ] Use only `SELECT 1` for the live query.
- [ ] If live API fails, switch to public-ready evidence; do not debug credentials on screen.
- [ ] Stop if any unexpected sensitive panel appears.

## After the demo

- [ ] Remove Workflow API URL/key from the process environment.
- [ ] Zero the BSTR pointer when present.
- [ ] Close the terminal and browser session.
- [ ] Record whether the demo was live or fallback evidence; do not alter verification reports.
