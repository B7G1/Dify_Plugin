# Report Archive Policy

No report was deleted for v1.0. Existing dated phase reports and the repository's established `archive/` / `reports/root_reports/过往报告/` areas retain historical and superseded material.

When a future report duplicates current meaning:

1. Keep the latest approved document in its canonical category.
2. Move the older document and its assets together into a dated archive directory.
3. Add a pointer to its replacement.
4. Do not move machine verification evidence or break relative HTML asset paths.

The v1.0 cleanup intentionally uses indexes rather than bulk moves because many historical HTML pages and reports contain relative links.
