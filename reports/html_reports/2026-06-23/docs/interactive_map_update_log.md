# Interactive Map Update Log

## 本次重构

- 重写 Internal Call Graph：HTML 节点由 CSS Grid 排列，SVG overlay 在运行时用每个节点的 `getBoundingClientRect()` 自动生成贝塞尔边。
- 重写复现区：左侧为文件/平台对象预览，右侧为步骤滚轮、当前步骤说明和 Change Log；下方保留联动的 Diff 编辑器。
- 文件树增加缩进、引导线、文件/目录图标、NEW/MOD 标签和平台对象分区。
- 文件、Change Log、步骤和调用图节点均可打开详情与代码 Diff；悬停显示来源、作用、关联及运行影响。
- 平台步骤改为 Dify / Docker 模拟场景：容器、PostgreSQL 网络、插件卡片、Provider 表单、Workflow 节点与待补测运行结果各有独立预览。
- 源码浏览器与代码编辑器合并为同一近邻开发工作区；关键 Python/YAML 文件显示完整关键内容，新增/修改行保持颜色标记。

## 为什么不继续修旧 CSS

旧图依赖 `nth-of-type`、绝对坐标和手工路径，节点或提示元素一增减便会改变选择器匹配和连线位置。新图没有这些定位规则：布局交给 Grid，边端点由浏览器实际测量。

## 如何使用

1. 在右侧步骤滚轮选择 Step 1–12；预览、Change Log 和编辑器同时更新。
2. 点击 Change Log 定位到关联文件和 Diff。
3. 点击/悬停文件树或调用图节点查看来源、作用、调用关系和运行影响。
4. 调整浏览器窗口大小，调用图边会自动重新计算。
